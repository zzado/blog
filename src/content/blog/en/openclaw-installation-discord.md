## Background

I wanted more than just chatting with AI on a website. I wanted to **summon AI right from my messenger** â€” reading files, writing code, making Git commits, all on command. That's exactly what [OpenClaw](https://docs.openclaw.ai/) does.

OpenClaw is a self-hosted gateway that connects messaging apps like WhatsApp, Telegram, Discord, and iMessage to AI agents. Since it runs on your own machine, your data stays with you and customization is unlimited.

I needed a machine that could run 24/7, so I picked up a **Mac mini** (M4) â€” got it for around $550 with a student discount! Low power consumption, compact size, and macOS being the most stable platform for OpenClaw made it a perfect fit.

## Process

### Step 1: Prerequisites

You need two things to run OpenClaw:

- **Node.js 22 or later**
- **AI model access** (Anthropic API key or Claude subscription)

```bash
node --version
# v22.x.x or higher
```

![Node.js version check](/blog/images/blog/node-version.png)

AI model options include:

- **[Claude Code](https://claude.ai/code)** â€” $100/mo, for developers (includes API credits, recommended)
- **[Claude Pro](https://claude.ai/pro)** â€” $20/mo, for general users
- **[Anthropic API](https://console.anthropic.com/)** â€” Pay-as-you-go

### Step 2: Install OpenClaw

One line does it all:

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

Verify the installation:

```bash
openclaw --version
```

![OpenClaw version check](/blog/images/blog/openclaw-version.png)

### Step 3: Onboarding Wizard

This is the key part. The `openclaw onboard` command walks you through all initial setup interactively.


### Step 4: Create a Discord Bot

Create your Discord bot at the [Discord Developer Portal](https://discord.com/developers/applications).

**Steps:**

1. **New Application** â†’ Enter app name (e.g., "OpenClaw Bot") â†’ **Create**
2. Left menu **Bot** â†’ **Reset Token** â†’ Copy the token
3. Under **Privileged Gateway Intents**:
   - âœ… **Message Content Intent** â€” **Required!** Without this, the bot can't read messages
   - âœ… **Server Members Intent** â€” Recommended (needed for user name lookups)
4. **OAuth2 â†’ URL Generator**:
   - Scopes: `bot`, `applications.commands`
   - Bot Permissions: `View Channels`, `Send Messages`, `Read Message History`, `Embed Links`, `Attach Files`, `Add Reactions`
5. Use the generated URL to invite the bot to your server

> âš ï¸ Forgetting **Message Content Intent** will cause a `Used disallowed intents` error. Don't skip it!

### Step 6: Verify & Test

Check Gateway status:

```bash

```

Open the web dashboard:

```bash

```

Navigate to `http://127.0.0.1:18789/` to access the Control UI, where you can chat with the AI directly.

Now try mentioning the bot in Discord:

```

```

If the bot responds, the integration is complete! You can also DM the bot, though first-time contact requires **pairing authentication**:

```bash
openclaw pairing approve discord <code>
```
